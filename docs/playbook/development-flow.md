# Development Flow

## Feature Delivery Steps
1. Feature Brief
- Define user story, scope/non-scope, and UX acceptance criteria.

2. Design Spec
- Confirm route impact, state ownership, repository changes, and edge cases.

3. Implementation
- Add/update route, feature modules, hooks/store, tests, and docs.

4. Verification
- Run typecheck, unit/integration tests, e2e smoke, and build.

5. Review Handoff
- Include screenshot diffs, update ADR if architecture changed, and create follow-up issues.

## Definition of Done
- Behavior complete
- Tests complete
- Docs complete
- CI green
